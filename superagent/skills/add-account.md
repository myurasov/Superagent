# Superagent add-account skill

## 1. Collect the account's data

Use `AskQuestion` for the kind question; inline for the rest.

- **Name** (e.g. "Chase Total Checking", "Fidelity 401(k)", "PG&E electric").
- **Institution** (bank / brokerage / utility / subscription provider).
- **Kind** (single-select):
  - checking
  - savings
  - credit_card
  - brokerage
  - retirement (401k / 403b / IRA / Roth / pension)
  - loan
  - mortgage
  - utility
  - insurance
  - subscription
  - membership
  - wallet (Apple Pay / Google Pay reference)
  - hsa / fsa / hsa_invest
  - other
- **Number last 4** (for matching transactions). NEVER ask for the full number.
- **Vault ref** — where the full credential lives. Ask the user for the password-manager item identifier (e.g. `1Password://Personal/chase-checking`, `keychain:bofa-savings`). Optional but encouraged.
- **URL** (login URL).
- **Primary holder** — `self` | `joint` | `spouse` | `child:<name>` | other.
- **Currency** (default `config.preferences.money.currency`).
- **Opened date** (ISO 8601 date or null).
- **Beneficiaries** (relevant for retirement / life insurance — list of names).
- **Low-balance threshold** (for checking / savings; defaults to `config.preferences.money.low_balance_default`).
- **Notes** — APR, terms, special instructions.

Auto-derive **id** = `<kind>-<institution-short>-<last4 or other discriminator>` (e.g. `checking-chase-1234`, `401k-fidelity`, `electric-pge`).

## 2. Check for duplicates

Search `_memory/accounts-index.yaml.accounts[]` by `id` and by `(institution, kind, number_last4)`. Confirm if similar.

```yaml
  notes: "<notes>"

```

## 4. Cross-link

Ask:
- **"Are any existing bills paid from this account?"** — for each yes, set `linked_bills` here AND `bills.yaml.<bill>.pay_from_account = <account-id>`.
- **"Are any existing subscriptions charged to this account?"** — same for subs.

## 5. Optional — wire to a finance ingestor

If `_memory/data-sources.yaml` has Plaid or Monarch enabled:

> "Want to link this account to **Plaid / Monarch** so transactions get auto-ingested? (yes / later / never)"

On `yes`:
- For Plaid: invoke `python3 superagent/tools/ingest/plaid.py --link-account <id>` (which walks the user through the link flow and sets `plaid_item_id`).
- For Monarch: invoke `python3 superagent/tools/ingest/monarch.py --link-account <id>`.

## 6. Update Domain

If `domain` for this account is `finance` (the typical case), append an entry to `Domains/Finance/history.md`:

```markdown

<Kind> at <Institution>; ending <last4>; <URL>.

```
