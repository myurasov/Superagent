# Superagent health-log skill

> **Pre-flight (lazy-materialization contract — `contracts/domains-and-assets.md` § 6.4a)**:
> This skill writes to `Domains/Health/{info,status,history}.md`. Before the
> first such write in this turn, run:
>
>     uv run python -m superagent.tools.domains ensure health
>
> Idempotent — near-no-op when `Domains/Health/` already exists; materializes
> the 5-file scaffold otherwise.

## 1. Classify the event

Use `AskQuestion` if not obvious from the trigger:

- symptom
- vital reading
- medication start
- medication stop
- medication change
- visit
- vaccination
- lab result

Also ask **for which member**: self | name (spouse / kid / pet) — default `self`.

## 2. Sub-flow per kind

### Symptom

Append to `_memory/health-records.yaml.symptoms[]`:

```yaml
- timestamp: <now>
  member: "<member>"
  symptom: "<short description>"
  severity: <0-10>
  triggers: <list>
  what_helped: <list>
  notes: "<free text>"
```

If `severity >= 7` OR symptom is in a known-urgent list (chest pain, severe headache, vision changes, etc.): **suggest urgent care** and create a P0 task "Decide whether to seek care for <symptom>".

### Vital reading

Append to `_memory/health-records.yaml.vitals[]`:

```yaml
- timestamp: <now>
  member: "<member>"
  kind: blood_pressure | heart_rate | weight | temperature | glucose | spo2 | sleep_score | hrv | other
  value: <numeric or compound>
  unit: "<unit>"
  source: "manual"
  notes: "<text>"
```

Flag if outside normal range (BP > 140/90 OR < 90/60; HR > 100 resting; temp > 100.4°F; glucose > 180 or < 70 fasting; SpO2 < 92).

### Medication start / stop / change

For **start**: append a row to `medications[]` with `status: active`. Compute `next_refill_due` if `refills_left` > 0 and `schedule` implies a daily-or-more cadence.

For **stop**: set the existing row `status: discontinued`, `end_date: today`. Ask for reason; capture in `notes`.

For **change**: ask which field (dose / schedule / pharmacy). Update the row. Append a one-line note ("changed to 40mg from 20mg as of <date>").

After any med change, update `Domains/Health/info.md` § Key Facts (the medications block) and `Domains/Health/status.md` § Recent Progress.

### Visit

Delegate the appointment side to `appointments mark-complete <id>` if a matching appointment exists. Otherwise:

1. Ask: provider (offer `add-contact` if missing), date, reason, findings, plan, medications_changed (yes/no).
2. Append to `_memory/health-records.yaml.visits[]`.
3. Append H4 entry to `Domains/Health/history.md`.
4. For each follow-up, create a P2 task.

### Vaccination

Append to `_memory/health-records.yaml.vaccines[]`:

```yaml
- member: "<member>"
  vaccine: "<name>"
  date: <date>
  dose_in_series: "<e.g. 1/2 or booster>"
  provider: "<contact id or free text>"
  lot: "<lot>"
  site: "<arm>"
  notes: "<text>"
```

Update `Domains/Health/info.md` § Key Facts (the immunization block).

### Lab result

Append to `_memory/health-records.yaml.results[]`:

```yaml
- date: <date>
  member: "<member>"
  kind: lipid_panel | cbc | a1c | tsh | imaging | biopsy | other
  ordered_by: <contact-id>
  abnormal: <bool>
  summary: "<one paragraph — the user's takeaway>"
  path: "<relative path to PDF; vault-grade lab PDFs go under Sources/medical/<member>/ via `add-source --to-domain health` (path is suggested; user picks); working scans go in Domains/Health/Resources/<member>/>"
  notes: "<text>"
```

If `abnormal: true`: create a P1 task to "Discuss <kind> result with <ordered_by>" with due date 7 days out.
