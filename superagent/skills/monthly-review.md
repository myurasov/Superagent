# Superagent monthly-review skill

Compute the **trailing calendar month** (1st of last month through end of last month, OR last 30 days if user prefers; honor `config.preferences.monthly_review_window` if set).

## 1. Run monthly-cadence ingestors

For sources with `schedule: monthly`, run them now (notably `photos`, `gmaps_timeline` which are heavy).

## 2. Subscription audit

Read `_memory/subscriptions.yaml`:

For each row with `status: active`:
- Compute `last_used_age_days` = today minus `last_used`.
- If `last_used_age_days > config.preferences.surfacing.subscription_unused_days` → set `audit_flag: true`.
- If `last_used` is null AND `started_at` > 90 days ago → flag `unsure` (no usage signal but still being charged).
- Compute trailing 12-month cost (`amount` × per-cadence multiplier).

Surface a table of audit candidates:

```
| Subscription | Cost / yr | Last used | Verdict | Action |
|---|---|---|---|---|
| Netflix | $216 | 4 days ago | keep | — |
| Adobe CC | $720 | 92 days ago | consider-cancelling | cancel? downgrade? |
| Spotify Family | $204 | yesterday | keep (shared with N people) | — |
```

For each `consider-cancelling`, ask the user **keep / cancel / unsure / delegate-to-replaceable**.

On `cancel`: open `subscriptions.yaml`, set `status: cancelled`, `cancelled_at: now`, prompt for `cancellation_reason`.

## 3. Document expiration scan

Read `_memory/documents-index.yaml`:

- Expiring in next 30 days → P1 task auto-created.
- Expiring in next 60 days → P2 task suggested.
- Expiring in next 90 days → surface but don't task.

For each, format: `{title} for {for_member} — expires {expires_at}`.

## 4. Vehicle / home maintenance windows

Read `_memory/assets-index.yaml`:

- For every asset's `maintenance` rows, compute `next_due`.
- Surface those due in next 60 days, grouped by domain.
- For each, suggest scheduling action.

## 5. Financial month recap

If finance ingestors are enabled:

1. Total inflows / outflows for the month.
2. Top 5 spending categories with month-over-month delta.
3. Savings rate ((inflows - outflows) / inflows) and trend vs 3-month average.
4. Net-worth snapshot if `accounts-index.yaml` has balance ingestion (Plaid / Monarch can pull balances).
5. Anomalies (any single transaction > 1.5× P95 of last 90 days for that category).
6. **Tax-relevant captures**: any transactions in tax-deductible categories (charitable, medical, business, education) — auto-tagged for end-of-year compilation.

Skip with explanatory note if no finance ingestor.

## 6. Health pass

Read `_memory/health-records.yaml`:

- **Overdue cleanings / vaccines / screenings**: derive from `vaccines[]` (CDC schedule for kids; annual flu / Tdap booster for adults; per-condition cadence) and from `appointments.yaml.history` (annual physical, dental cleaning at 6mo, eye exam yearly).
- **Med refills due in next 30 days**.
- **Active conditions with no visit in 6 months** for any chronic-care relationships.
- **Lab results flagged abnormal** that haven't been follow-up booked.

For each overdue item, suggest `add-appointment` or `appointments find-next-slot`.

## 7. Domain hygiene

For each domain in `domains-index.yaml` with `status: active`:
- Read its `history.md` last entry date.
- If > 60 days ago AND `_memory/todo.yaml` has zero open tasks for this domain → surface "Is **<domain>** still active?".

User can mark a domain `paused` (kept around but not surfaced) or trigger an archive proposal (`doctor` skill picks up later).

## 7d. Domain suggestions (detection-driven)

Per `contracts/domains-and-assets.md` § 6.4b, run the domain detector and
surface candidates the user hasn't already accepted / declined / deferred.

```
uv run python -m superagent.tools.domain_detector run --json
```

If the JSON list is non-empty, delegate to the `domain-suggest` skill to
batch-ask the user (yes / not now / never) per candidate. Each accepted
candidate routes to `add-domain` with the pre-filled name + scope. Each
"not now" defers 90 days; each "never" suppresses permanently.

This is the periodic surface — the ambient mid-conversation surface (per
§ 6.4b) runs at any time during normal turns when the agent observes a
strong cluster signal.

## 7b. Project archival proposals

Read `_memory/projects-index.yaml.projects[]`:

- For each project with `status: completed` AND `completed_date > config.preferences.projects.archive_after_days` (default 90) ago → propose archive (move to `Archive/<YYYY-MM>/Projects/<slug>/`).
- For each `recurring` project just completed (status flipped this month), surface the auto-created next-cycle row for confirmation: "I created `<next-slug>` for next year. Charter inherited from this year. Look right?".
- For each project in `planning` for > 90 days with no chartering activity → prompt "still planning, or drop?".

## 7c. Sources hygiene

Read `_memory/sources-index.yaml`:

- Sources with `read_count: 0` AND `added > 180 days ago` → surface "haven't used this in 6 months. Keep, or move to Archive/?".
- Sources with broken `path` (file no longer exists) → surface as `broken-link`.
- `Sources/_cache/` total size + LRU eviction stats since last monthly-review.

Note: per the immutability contract (§ 15.2), `doctor` cannot delete from `Sources/` automatically. The user does the deletion manually if they choose; this section just surfaces candidates.

## 8. Personal-signal rollup

Read `_memory/personal-signals.yaml`:
- Count of new signals this month (by category).
- Surface 1–3 signals where `surfaced_count == 0` for review.

## 10. Update memory

1. `_memory/context.yaml.last_runs.monthly_review` → now.
2. Append to `_memory/interaction-log.yaml`.
3. Append the H4 entry to `Domains/Self/history.md`.
4. Persist any subscription `status` flips, document tasks, health-followup tasks created.
