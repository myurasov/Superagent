# Superagent vehicle-log skill

## 1. Resolve the vehicle

If only one vehicle is `status: active` in `assets-index.yaml`, default to it. Otherwise ask which vehicle (by name / slug).

## 2. Classify the event

- service (any maintenance work)
- fuel-up
- mileage reading
- repair (unscheduled)
- accident
- registration renewal
- inspection
- insurance renewal

## 3. Sub-flow per kind

### Service

Ask:
- **Service kind** (oil change, tire rotation, brakes, transmission, coolant, …).
- **Date** (default today).
- **Mileage** at service.
- **Cost**.
- **Performed by** (contact id from `contacts.yaml` if a known mechanic, else free text — offer `add-contact`).
- **Notes** (issues found, recommended next-time).

Update:
- `assets-index.yaml.<vehicle>.maintenance` — for the matching maintenance kind, set `last_done: <date>`, recompute `next_due`.
- Append H4 entry to `Domains/Vehicles/history.md`:

  ```markdown
  #### <YYYY-MM-DD> — <Vehicle name>: <service kind>

  Mileage <mileage>. Cost $<cost>. Performed by <mechanic>.

  <Notes>.
  ```
- If a P-task existed for the upcoming service, mark it done.
- Add a new P2 task for the next service if `next_due` is within 90 days.

### Fuel-up

Ask: date, mileage, gallons, cost. Optionally compute MPG if previous fuel-up exists.

Append a one-line entry to `Domains/Vehicles/history.md`:

```markdown
#### <YYYY-MM-DD> — <Vehicle name>: fuel-up

Mileage <mileage>. <Gallons> gal at $<cost> (MPG: <mpg if computable>).
```

(Optionally: append to a per-vehicle `Domains/Vehicles/Resources/<slug>/fuel-log.csv` for trend analysis. The CSV is a working file — not a vault document — so it lives in `Resources/`, not `Sources/`.)

### Mileage reading

Ask: date, mileage. Update an `assets-index.yaml.<vehicle>.notes` "current mileage" if you maintain one, AND append a one-line history entry.

### Repair (unscheduled)

Ask: date, mileage, what broke, what was fixed, cost, performed by. Append H4 entry to history. Add a P2 task to "Watch for recurring <issue>" if user wants to track.

### Accident

Ask: date, location, severity (minor / fender-bender / serious), other parties (offer `add-contact`), claim filed (yes/no), insurance contact.

Append H4 entry to history with full details. Auto-create a P0 task "File / follow up on insurance claim for <date> accident" if claim filed. Add to `documents-index.yaml` for any photos / police report scans (if the user has them).

### Registration / inspection / insurance renewal

Ask: date renewed, expires_at (next renewal), cost, registration sticker / inspection certificate location.

Update `documents-index.yaml` for the registration / inspection card; add or refresh `expires_at` for the matching document.

Update `bills.yaml.<bill>.history[]` for the renewal payment.

Add to history.

## 4. Sync downstream

- Append to `interaction-log.yaml`:
  ```yaml
  - timestamp: <now>
    type: maintenance
    subject: "<vehicle>: <kind>"
    summary: "<short>"
    related_domain: "vehicles"
    related_asset: "<vehicle slug>"
  ```
- Update `Domains/Vehicles/status.md` § Recent Progress and § Next Steps.

## 5. Confirm

```
Logged <kind> for <vehicle> on <date>.
Mileage: <mileage>. Cost: <cost>.
Next service in: <days> days / <miles> miles.
```
