# Superagent add-document skill

## 1. Collect the document's data

- **Title** (e.g. "Passport — self", "Auto insurance — Camry").
- **Kind** (single-select): passport | drivers_license | state_id | social_security | birth_certificate | marriage_certificate | divorce_decree | death_certificate | naturalization | green_card | visa | deed | title | mortgage_doc | lease | will | trust | power_of_attorney | living_will | medical_directive | insurance_policy | warranty | receipt | tax_return | w2 | 1099 | k1 | property_tax | school_record | medical_record | vaccine_record | prescription | diploma | certification | contract | nda | other.
- **For member** — "self" | name | "household".
- **Issuer** (e.g. "U.S. Department of State", "California DMV", "Geico").
- **Number** — last 4 digits ONLY for sensitive kinds (passport / SSN / DL); full number ok for non-sensitive (warranty / receipt).
- **Vault ref** — where the full number / scan lives (e.g. "1Password://Personal/passport-self").
- **Issued date** (ISO 8601).
- **Expires at** (ISO 8601 — drives expiration surfacing).
- **Path** — workspace-relative to the digitized file (offer to file via `Inbox/` workflow if not yet digitized).
- **Physical location** — where the original is stored (e.g. "fireproof safe", "safe deposit box at Wells Fargo Hayward", "office filing cabinet drawer 2", "with parent in <city>").
- **Domain** — auto-suggest based on `kind`:
  - passport / drivers_license / state_id / social_security / birth_certificate / marriage_certificate → `family` (or `self`)
  - deed / mortgage_doc / lease → `home`
  - title → `vehicles`
  - will / trust / power_of_attorney / living_will / medical_directive → `family` (or `self`)
  - insurance_policy → `finance`
  - warranty → the asset's domain
  - tax_return / w2 / 1099 / k1 / property_tax → `finance`
  - school_record / diploma → `family` or `career`
  - medical_record / vaccine_record / prescription → `health`
  - certification → `career`
- **Notes** — anything else (e.g. "include for international travel").

Auto-derive **id** = `doc-<kind>-<for_member>` (e.g. `doc-passport-self`, `doc-insurance-camry`).

## 3. Expiration guard

If `expires_at` is set, surface immediately:
- < 30 days → P1 task auto-created ("Renew <Title>").
- 30–60 days → P2 task suggested.
- 60–90 days → just echo, no task.

## 4. File the digitized document

If `path` is in `workspace/Inbox/`, offer:

> "Want me to move this to `Sources/<category>/<filename>`? (yes / no — you can also pick a different path)"

On `yes`: move the file via `add-source --to-domain <inferred>` (which routes it correctly + adds a row to the relevant domain's `sources.md` + refreshes `_memory/sources-index.yaml`). Update `documents-index.yaml.<doc>.path` to the new `Sources/<category>/<...>` location.

This skill is being progressively superseded by `add-source` — the canonical capture skill for any document. `add-document` survives for the structured-metadata fields specific to vital records (passport / license / etc.) that need expiration tracking; the file-placement side delegates to `add-source`.

**Payment receipts**: when `kind: receipt` and the receipt represents a payment (vs. a purchase warranty), the canonical capture path is `contracts/payment-confirmations.md`, NOT this skill. That contract decides destination (`Sources/<Domain>/` vs `Projects/<project>/Resources/`), cross-links to `bills.yaml` / `subscriptions.yaml` / `appointments.yaml`, and only invokes `add-document` for long-lived receipts that warrant a `documents-index.yaml` row (taxes, large medical, property, vehicle major service). Routine bill-payment confirmations skip the `documents-index` row.
