# Superagent add-bill skill

## 1. Collect the bill's data

- **Name** (e.g. "PG&E electric", "Geico auto insurance", "Wells Fargo mortgage").
- **Payee** (e.g. "Pacific Gas & Electric").
- **Account** (id from `accounts-index.yaml`; the utility / provider account; optional — null if untracked).
- **Pay-from account** (id from `accounts-index.yaml`; the bank / card that pays it).
- **Amount** (numeric — for variable bills, the last known amount).
- **Amount kind**: fixed | variable | usage-based.
- **Currency** (default from config).
- **Cadence**: monthly | quarterly | semi-annual | annual | one-shot | weekly | bi-weekly | semi-monthly | bi-monthly | irregular.
- **Due day** (e.g. 15 = the 15th; for weekly, day-of-week 1-7; for one-shot, leave null).
- **Auto-pay**: true / false.
- **Grace period** (days late before urgent; default 0).
- **Late fee** (numeric — for monthly-review surfacing).
- **URL** (where to pay manually).
- **Domain** (e.g. `home` for utilities; `vehicles` for auto insurance; `health` for medical bills).
- **Asset** (id from `assets-index.yaml`; for asset-linked bills like vehicle insurance).
- **Notes** — rate plan, contract end, promo expiration.

Auto-derive **id** = `bill-<slug>` (e.g. `bill-pge-electric`).

## 2. Compute next_due

If `cadence` and `due_day` are set, compute the next occurrence after today. Otherwise leave null and surface a warning.

## 3. Append the bill row

```yaml
- id: "<id>"
  name: "<Name>"
  payee: "<Payee>"
  account: <account-id or null>
  pay_from_account: <account-id or null>
  amount: <amount>
  amount_kind: "<kind>"
  currency: "<currency>"
  cadence: "<cadence>"
  due_day: <day>
  next_due: <computed>
  auto_pay: <bool>
  grace_period_days: <n>
  late_fee: <amount or null>
  url: "<url>"
  domain: "<domain>"
  asset: <asset-id or null>
  tags: []
  notes: "<notes>"
  history: []
  status: "active"
  created: <now>
  last_updated: <now>
```

## 4. Cross-link the account

If `pay_from_account` is set, append the bill id to `accounts-index.yaml.<acct>.linked_bills[]`.

## 5. Update domain status

Update `Domains/<Domain>/info.md` § Routines (add bullet: "Pay <Name> {cadence} on day {due_day}").

If `next_due` is within 7 days, also add a bullet to `Domains/<Domain>/status.md` § Next Steps.

## 6. Confirm

```

Next due: <date>.
Auto-pay: <yes/no>.
```
