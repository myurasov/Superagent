# Superagent add-appointment skill

## 1. Collect the appointment's data

- **Title** (short — e.g. "Annual cleaning", "30k mile service", "Annual physical").
- **Kind**: medical | dental | vision | mental_health | vet | mechanic | home_repair | beauty | school | legal | financial | personal | other.
- **Start** (ISO 8601 datetime with TZ).
- **End** (optional).
- **Location** (street + city; "telehealth"; "phone"; "<provider's office>").
- **Provider** — contact id from `contacts.yaml`. Offer to invoke `add-contact` if missing.
- **For member** — "self" | name (spouse, child, pet — pets are in `contacts.yaml` with role: pet).
- **Domain** — Health (medical / dental / vision / mental_health) | Pets (vet) | Vehicles (mechanic) | Home (home_repair) | Family | etc.
- **Asset** (id from `assets-index.yaml` — for vehicle service or appliance repair).
- **Purpose** — one-liner.
- **Prep notes** — what to bring, things to discuss, questions to ask.
- **Confirmation** — provider's confirmation number.
- **Recurrence** — if this is a recurring appointment, set `{ interval: <e.g. "annual">, n: 0 }`.

Auto-derive **id** = `appt-<YYYYMMDD>-<provider-slug or kind-slug>`.

## 3. Auto-create prep task (if applicable)

If `prep_notes` non-empty:

- Create P1 task in `todo.yaml`:
  - Title: "Prep for <Title> on <date>"
  - Due date: <start> minus 1 day
  - related_appointment: <id>
  - Description: "<prep_notes>"

## 4. Update domain rolodex

If `provider` is set, ensure the provider's row in `Domains/<Domain>/rolodex.md` exists; update `Last contacted` to the appointment date.

## 5. Update domain status

Add a bullet to `Domains/<Domain>/status.md` § Next Steps:

> "<Title> with {provider} on {start} ({location})"
