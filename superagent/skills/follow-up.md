# Superagent follow-up skill

## 1. Overdue and stale tasks

Read `_memory/todo.yaml`:

- **Overdue** — `status in [open, in_progress]` AND `due_date < today`.
- **Stale** — `status: open`, `due_date: null`, `created` > 14 days ago.
- **Completed but unconfirmed** — tasks `done` in last 3 days whose description includes "send", "confirm", "verify" — may need follow-through.

Sort by P0 → P1 → P2 → P3, then by days overdue.

## 2. Unanswered messages

For each message ingestor enabled (gmail, icloud_mail, outlook, imessage, slack, …):

Read the local mirror. Identify threads where:
- The user was asked a question (last message ends with `?` AND was sent TO the user) AND has no reply from the user in the thread within 3+ business days.
- The user sent a message with a clear ask ("please confirm", "let me know", "can you") and got no reply within 3+ business days.

Cap per source at 10 most relevant. Prioritize:
- From a contact in `rolodex.md` of an active domain.
- From a contact with `contact_cadence: monthly` or shorter.
- Subject contains an action verb or a domain-relevant keyword (medical / financial / urgent).

## 3. Unfulfilled commitments

Read `interaction-log.yaml` for the last 30 days. For entries with `action_items[]`:

- Cross-reference each action item against `todo.yaml`.
- If no matching task and no evidence of completion (no later entry mentioning it), flag as **missed commitment**.
- If a task exists but is overdue, flag as **overdue commitment**.

## 4. Stale appointments

Read `_memory/appointments.yaml`:

- Appointments with `status: scheduled`, `start` in next 14 days, AND `confirmation` empty AND created > 7 days ago — may need confirmation call.
- Appointments with `status: scheduled`, `start` in past, AND not yet `mark-complete`d → ask user to mark.

## 5. Compose the report

```
## Follow-up Report — <today>

### Overdue tasks (<count>)
- **P0** task-NNN: <title> — due <date>, <N> days overdue (<related_domain>)
…

### Stale tasks (<count>)
- **P2** task-NNN: <title> — created <date>, no due date

### Unanswered messages (<count>)
- **<sender>** (<domain>) — asked about <topic> (<date>, <N> days ago)

### Missed commitments (<count>)
- From <date> <interaction-log subject>: "<commitment>" — no task or reply found

### Stale appointments (<count>)
- <date> <title> with <provider> — no confirmation; consider calling

### Past appointments needing close-out (<count>)
- <date> <title> with <provider> — never marked complete; what happened?
```

Skip empty sections. For each item, suggest a concrete next action ("Reply with X", "Mark task done", "Call provider").

## 6. Offer to act

```
Want me to:
  - Draft replies for any unanswered messages?
  - Mark some tasks done?
  - Mark past appointments complete?
  - File the missed commitments as new tasks?
```

If the user picks items, hand off to `draft-email`, `todo`, `appointments mark-complete`, etc.
