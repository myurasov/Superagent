# Superagent ingest skill

The data-ingestion contract is in `contracts/ingestion.md`. This skill is the user-facing front-end.

## 1. Branch on intent

### `ingest` (no arguments) → status

1. Read `_memory/data-sources.yaml`.
2. For each source: id, kind, enabled, capture_mode, last_ingest, last_run summary, failure_streak.
3. Format a compact table:

   ```
   | Source | Status | Last run | Items pulled | Failures |
   |---|---|---|---|---|
   | gmail | scheduled (daily) | 4 hr ago | 23 (3 inserted) | 0 |
   | apple_health | manual | 2 days ago | 412 vitals | 0 |
   | strava | scheduled (daily) | 3 hr ago | 0 (no activity) | 0 |
   | plaid | scheduled (weekly) | 6 days ago | 87 transactions | 1 (auth refresh needed) |
   | … | disabled | never | — | — |
   ```

4. End with a one-line: "Total enabled: N / 23 supported. Run `ingest --setup` to enable more, `ingest --all` to refresh all enabled sources now."

### `ingest --setup` → discover + enable

Per `init` step 5, but standalone:

1. For each row in `data-sources.yaml` with `enabled: false`, run the source's `probe()` (lightweight presence check from `tools/ingest/<source>.py`).
2. Surface a table of `available` / `not detected` / `needs setup` per source.
3. For each `available` source, ask whether to enable. On `yes`: flip `enabled: true`, set `capture_mode: scheduled`, run a first ingest with the conservative `recency_window_days`.
4. For each `needs setup` source the user expressed interest in, point at `docs/data-sources.md#<source>` for the install steps.

### `ingest --all` → refresh all enabled

Run every enabled ingestor in parallel (where safe; some need to be serialized — see `tools/ingest/_serialization.yaml`).

Surface live progress; on completion, summary table.

### `ingest <source>` → refresh one

Run just the named source. Useful for targeted catch-up after fixing an auth issue.

### `ingest <source> --backfill`

Run with `backfill_window_days` instead of `recency_window_days`. Used for "I just enabled this; pull the last few years now that I trust it".

### `ingest <source> --reauth`

Re-authenticate the source. Delegates to the source's `reauth()` function (typically pops an OAuth flow or prompts for a token).

### `ingest <source> --disable`

Set `data-sources.yaml.<source>.enabled: false`. Doesn't delete the local mirror — just stops new ingest runs. Re-enable any time.

### `ingest <source> --status`

Per-source detail: last 5 runs, failure log, current config (recency window, max items, capture mode).

### `ingest --schedule`

Show the per-source cadence schedule. Edit via `config.yaml.preferences.ingestion_schedule.<source>`.

## 2. Source catalogue

The full catalogue (id, kind, what it ingests, where it writes, install steps, auth requirements) is in `docs/data-sources.md`. Brief summary of MVP-supported sources:

### Email + calendar
- **gmail** — Gmail (Google Workspace MCP). Writes to `interaction-log.yaml`, `appointments.yaml` (auto-detected from threads), `bills.yaml` (auto-detected from "your statement is ready"), `subscriptions.yaml` (auto-detected from "Welcome to" / receipts), per-domain `history.md`.
- **icloud_mail** — Apple Mail via iCloud MCP (IMAP). Same writes as gmail.
- **outlook** — Outlook personal mailbox. Same writes.
- **google_calendar** — events → `appointments.yaml` (when appointment-shaped) + per-domain `history.md`.
- **icloud_calendar** — same via CalDAV.
- **outlook_calendar** — same.

### Reminders + notes
- **apple_reminders** — `rem` CLI. Writes to `todo.yaml` (one-way: Reminders → Superagent).
- **apple_notes** — `osascript` against Apple Notes. Writes plain-text snapshots to `Domains/<inferred>/Resources/notes/` (working / browseable copies; not vault-grade).
- **obsidian** — vault path. Indexes notes; cross-links by frontmatter tags; can't write back unless `writes_upstream: true`.
- **notion** — official Notion API. Indexes pages / databases.

### Finance
- **plaid** — `plaid-cli` / `yapcli`. Writes to `_memory/transactions.yaml` (a derived index built by the ingestor), updates `accounts-index.yaml.<acct>` balances, reconciles into `bills.yaml` and `subscriptions.yaml`.
- **monarch** — `monarch-cli`. Same shape.
- **ynab** — YNAB API. Same.
- **csv** — generic bank CSV import. User points at a file; ingestor parses (supports common bank formats: Chase, BoA, Wells Fargo, Amex, Schwab, Fidelity).

### Health + wearables
- **apple_health** — `healthsync` CLI parses iPhone-side `export.zip`. Writes to `health-records.yaml.vitals[]`, `health-records.yaml.symptoms[]` (where applicable), per-pet vitals if Health share is set up.
- **whoop** — WHOOP MCP. Writes recovery / strain / sleep to `health-records.yaml.vitals[]`.
- **strava** — workouts → `Domains/Hobbies/history.md` AND `health-records.yaml.vitals[]` (HR averages).
- **garmin** — comprehensive (29+ tools); writes the same as the union of strava + whoop + apple_health.
- **oura** — sleep / readiness / activity → `health-records.yaml.vitals[]`.
- **fitbit** — coming-soon stub for now.

### Smart home + vehicles
- **home_assistant** — Home Assistant MCP. Writes daily anomaly notes (door-left-open, motion-where-shouldn't-be, sensor-out-of-range) to `Domains/Home/history.md`. Writes thermostat / energy / device-state snapshots to `Domains/Home/Resources/ha-snapshots/<date>.json` (working state — purgeable; not vault-grade).
- **smartthings** — Samsung-flavored equivalent.
- **tesla** — Tesla MCP. Writes mileage / charging / location to `assets-index.yaml.<vehicle>` AND `Domains/Vehicles/history.md`. Surfaces next-service signal when mileage crosses threshold.

### Communications
- **imessage** — `imessage-exporter` (read-only export of `~/Library/Messages/chat.db`). Writes to `interaction-log.yaml` (only "important contact" filter — not every message).
- **whatsapp**, **signal**, **telegram** — best-effort via Matrix bridge or vendor MCP. Conservative ingestion (read-only, opt-in per chat).
- **slack** — personal workspaces only. Writes to `interaction-log.yaml`.

### Files + media + location
- **photos** — `exiftool` over a Photos library export. Writes to `_memory/photo-locations.jsonl` (an append-only timeline) for "where was I on <date>" queries.
- **gmaps_timeline** — Google Takeout location-history JSON. Writes to `_memory/location-timeline.jsonl`.
- **icloud_drive / google_drive / dropbox / onedrive** — file-listing ingestors only. Index file metadata so the agent can answer "where did I save the X PDF?".

## 3. Sync downstream

After every run:
- Per-ingestor: writes to `data-sources.yaml.<source>.last_ingest` + `last_run`, appends to `ingestion-log.yaml`, appends to `interaction-log.yaml`.
- This skill: aggregates across runs and prints a final summary.

## 4. Privacy reminder

Surface, on `--setup` and on first `--all`:

> All ingested data stays local in `workspace/` (gitignored). Superagent does not phone home. Read-only by default — no source writes back upstream unless its `writes_upstream` flag is true (none in MVP).
