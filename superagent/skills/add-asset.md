# Superagent add-asset skill

> **Pre-flight (lazy-materialization contract — `contracts/domains-and-assets.md` § 6.4a)**:
> This skill writes to `Domains/<owning-domain>/Resources/<asset-slug>/` for
> working files (step 3) and updates `Domains/<owning-domain>/status.md` Next
> Steps when seeding maintenance (steps 4-5). Before either, run:
>
>     uv run python -m superagent.tools.domains ensure <owning-domain>
>
> Idempotent. The owning domain is the `domain` field on the asset row
> (e.g. `vehicles`, `home`, `assets`).

## 1. Collect the asset's data

Use `AskQuestion` (one batch where reasonable, otherwise short follow-ups):

- **Name** (e.g. "Blue Camry", "Living-room Samsung TV", "Mom's pearl necklace").
- **Kind**: vehicle | appliance | electronics | jewelry | instrument | tool | collectible | furniture | art | other.
- **Sub-kind** if vehicle: car | motorcycle | bicycle | boat | RV | truck | scooter | other.
- **Domain**: which domain owns this? (default heuristic: vehicle → `vehicles`; appliance → `home`; jewelry → `home` or `family`; tool → `home` or `hobbies`; instrument → `hobbies`; other → ask).
- **Make / Model / Year**: free text.
- **Color** (if useful for identification).
- **Serial** (electronics / appliances) OR **VIN** (vehicles).
- **Purchase date** (ISO 8601 date).
- **Purchase price** (numeric).
- **Warranty expiration** (date or null).
- **Insurance** (one line: carrier + policy ref).
- **Location** (where it lives — e.g. "garage", "kitchen", "safe deposit box").
- **Notes** — anything else worth remembering.

Auto-derive **slug** = `<kind>-<color-or-make>-<short-model>-<year>` lowercased, hyphenated, e.g. `car-blue-camry-2018`. Confirm.

## 3. Create per-asset sub-folders (optional)

If the user has receipts / scans / titles / warranty PDFs to file (vault-grade documents):

1. **Source documents** — invoke `add-source --to-domain <Domain> --asset <asset-slug>` for each. The skill will SUGGEST a path like `workspace/Sources/<category>/<asset-slug>/` (you can override) and add a row to `Domains/<Domain>/sources.md`. Source documents NEVER go under `Domains/<X>/` directly.
2. **Working files** — for context photos (the serial-number sticker, the leak that started this), per-asset `workspace/Domains/<Domain>/Resources/<asset-slug>/` is the right home. Created lazily.

Do NOT create either folder if there's nothing to put in it.

## 4. Recurring maintenance — optional but high-leverage

For common asset kinds, propose default maintenance schedules:

| Kind / sub-kind | Default maintenance |
|---|---|
| car | oil change every 5000 mi or 6 mo; tire rotation every 7500 mi; brake inspection annually; registration renewal annually; insurance renewal per policy |
| motorcycle | oil change every 3000 mi; chain lube monthly; registration renewal annually |
| bicycle | tune-up every 1000 mi or annually |
| HVAC | filter change every 3 mo; service annually (spring) |
| furnace | service annually (fall) |
| water heater | drain annually; anode rod check every 3 yr |
| refrigerator | water filter change every 6 mo; coils clean annually |
| dishwasher | filter clean monthly; deep clean annually |
| washing machine | clean drum monthly |
| dryer | vent clean annually |
| smoke detector | battery change annually; replace unit every 10 yr |
| fire extinguisher | inspect annually; replace per manufacturer |
| pool / spa | per-week chemistry; annual service |
| septic system | pump every 3-5 yr |
| roof | gutter clean spring + fall; inspection every 2 yr |
| firearm | clean per use; safe-store check annually |

Ask: *"Want me to add the standard maintenance schedule for <kind>? (yes / customize / skip)"*

On `yes` or `customize`: append rows to `assets-index.yaml.<asset>.maintenance[]` and compute `next_due` for each. ALSO add corresponding rows to `_memory/bills.yaml` with `kind: maintenance`, `cadence: matching the maintenance interval`, so it shows up in cadence skills. Optionally add a P2 task to `todo.yaml` for the soonest one.

## 5. Update domain status

If the new asset has open maintenance items in next 30 days, update `Domains/<Domain>/status.md` § Next Steps with one bullet per item.

Append an entry to `Domains/<Domain>/history.md`:

```markdown
#### <YYYY-MM-DD> — Acquired <asset name>

<Kind> · <make> <model> <year> · <color> · purchased <date> for <price>.
Stored at <location>.
Maintenance schedule: <one-line summary>.
```
