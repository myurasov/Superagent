# Superagent home-maintenance skill

> **Pre-flight (lazy-materialization contract — `contracts/domains-and-assets.md` § 6.4a)**:
> This skill writes to `Domains/Home/{history,status}.md` when logging
> completion. Before the first such write in this turn, run:
>
>     uv run python -m superagent.tools.domains ensure home
>
> Idempotent. Pure list-due reads do not need the ensure call.

## 1. Branch on intent

### List due

1. Read `_memory/assets-index.yaml`.
2. Filter to assets with `domain: home` AND `status: active`.
3. For each asset, iterate its `maintenance[]` rows.
4. Surface items with `next_due <= today + config.preferences.surfacing.maintenance_lookahead_days`.
5. Group by urgency:
   - **Overdue**: `next_due < today`.
   - **Due in next 7 days**.
   - **Due in next 30 days**.
6. Format: `{asset} → {kind} (due {next_due}, last done {last_done})`.

### Log completion

User said "I just changed the HVAC filter" / "I just had the gutters cleaned":

1. Resolve asset + maintenance kind. Confirm if multiple match.
2. Ask: date (default today), cost (optional), notes.
3. Update the maintenance row:
   - `last_done: <date>`
   - Recompute `next_due` from `last_done + interval_days`.
4. Append H4 entry to `Domains/Home/history.md`.
5. Mark any matching P-task done.

### Schedule

User asked to add a maintenance kind to an asset:

1. Resolve asset.
2. Ask: kind, interval_days.
3. Append a row to `assets-index.yaml.<asset>.maintenance[]`:
   ```yaml
   - kind: "<kind>"
     interval_days: <n>
     interval_miles: 0
     last_done: <if known>
     next_due: <computed>
   ```
4. If `next_due` is within 30 days, add a P2 task.

## 3. Sync downstream

After any operation:
- Update `Domains/Home/status.md` § Next Steps with the next 1-3 due items.
- Append to `interaction-log.yaml`.

## 4. Cost rollup

`home-maintenance --cost <year>` summarizes all logged costs for the year by category and asset. Useful for tax-time and home-sale price-history records.
