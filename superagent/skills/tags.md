# Superagent tags skill

Implements superagent/docs/_internal/ideas-better-structure.md item #11. Backed by `_memory/tags.yaml`.

## 1. Branch on intent

### List (default)

1. Read `_memory/tags.yaml.tags[]`.
2. Sort by `uses_count` desc.
3. Render compact table:
   ```
   | tag | uses | category | description |
   |---|---|---|---|
   ```

### Search

User asked "show me everything tagged X":

1. Resolve canonical tag id (use aliases).
2. Walk every entity-shape YAML under `_memory/`. For each row, check `tags: [..]`.
3. Group results by file. Output:
   ```
   ## bills.yaml (3 matches)
   - bill:pge-electric (PG&E electric)
   - bill:water-utility (Water utility)
   ...
   ```
4. Update `tags.yaml.<tag>.last_seen` to today.

### Add (auto-register)

When the user (or another skill) introduces a new tag and `config.preferences.tags.auto_register` is true:

1. Append a row to `tags.yaml.tags[]`:
   ```yaml
   - id: "<canonical-slug>"
     aliases: []
     description: ""
     category: ""
     uses_count: 1
     first_seen: <today>
     last_seen: <today>
     created_by: "<skill or user>"
   ```
2. Optionally prompt for description + category right then.

### Rename

User asked to rename `<old>` to `<new>`:

1. Validate: `<new>` either doesn't exist, or exists with no overlap (we'll just absorb).
2. Walk every entity-shape YAML; replace `<old>` with `<new>` in every `tags` list.
3. Update `tags.yaml`: append `<old>` to `<new>.aliases`; delete `<old>` row.
4. Update `world.yaml` edges (kind: `tagged`) — call `tools/world.py rebuild` once at end.

### Merge

User asked to merge `<keep>` and `<absorbed>`:

1. Same as rename, but record the merge in `_memory/tags.yaml.<keep>.notes`.
2. Append the absorbed tag's aliases to the kept tag's `aliases`.

### Recount

Always-safe maintenance:

1. Walk every entity-shape YAML; count uses per canonical tag.
2. Update `tags.yaml.<tag>.uses_count`.
3. Surface tags with `uses_count == 0` for cleanup confirmation.

## 2. Strict canonical mode

If `config.preferences.tags.strict_canonical` is true:

When any other skill writes a tag NOT in `tags.yaml`, that skill must REFUSE the write OR auto-register the new tag with `created_by: <skill>` AND surface a one-line warning so the user can curate it later.

Default in MVP is `strict_canonical: false` (lenient).

## 3. Sync downstream

After any rename / merge / add:
- Update `world.yaml` (call `tools/world.py rebuild`).
- Update `interaction-log.yaml`.
- Recompute `tags.yaml.<tag>.uses_count`.
