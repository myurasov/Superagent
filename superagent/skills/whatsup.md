## 1. Establish the time baseline

Open `_memory/context.yaml` and read `last_check`. If missing or null, treat baseline as **start of today** in the user's local timezone and state that assumption.

## 2. Bills due (next 7 days + overdue)

Read `_memory/bills.yaml`:

- Compute `next_due` for any row where `next_due` is null (use `cadence` + `due_day` + `history[-1]`).
- Filter to rows with `status: active` and `next_due <= today + 7 days`.
- Split into:
  - **Overdue** (`next_due < today` AND not in `history` for the current cycle).
  - **Due today**.
  - **Due in 1ŌĆō7 days**.

For each, format: `{name} {amount currency} ŌĆö {next_due} ({manual / auto-pay})`.

## 3. Today's appointments

Read `_memory/appointments.yaml`:

- Filter `status: scheduled` AND `start` is on today (local).
- For each: `HH:MM ŌĆö {title} with {provider} ({location})`.
- Flag any with `prep_done: false` and `prep_notes` non-empty.

Also list the **next 2** upcoming appointments after today (date + title) for context.

## 4. Important dates (today / tomorrow / next 7)

Read `_memory/important-dates.yaml`:

- Recompute `next_occurrence` for each row.
- **Today**: `{title} ({recurrence_year_count} years)`.
- **Tomorrow**: same.
- **Next 7 days**: list (with date prefix).

If a date in the gift-shopping window has empty `gift_log` for this year, surface the prompt: *"Want a gift idea for {title}?"* (offers to draft via `important-dates suggest-gift <id>`).

## 5. Tasks (P0 + overdue)

Read `_memory/todo.yaml`:

- **P0 open**: list (id, title, due, related_domain).
- **P1 open AND overdue** (`due_date < today`).
- **P1 open AND due in next 3 days**.

If overdue count > 0, end with:

> **<N> overdue tasks** ŌĆö run `triage-overdue` to force a decision on each.

## 6. Alerts from ingested data (since last_check)

Read `_memory/ingestion-log.yaml`:

- Find ingest runs with `started_at >= last_check`.
- Aggregate notable destination_summary fields:
  - New `interaction_log` entries (count + top 3 if any flagged as action-needed).
  - New `bills` rows (potential auto-detected recurring charge).
  - New `appointments` rows (calendar-detected).
  - New `health-records` events (med refill due, lab result tagged abnormal).
  - Any per-source `errors` not yet acknowledged.

Surface anything that warrants attention, max 5 items.

## 7. Open signals (Supertailor + Superagent)

Read `_memory/action-signals.yaml`:

- T = count of `target: tailor` AND `status: captured`.
- S = count of `target: superagent` AND `status: captured`.

If T > 0 AND last `supertailor-review` > 30 days ago: surface `Consider: supertailor-review ŌĆö N unprocessed framework signals.`

If S > 0: surface `Consider: drain pending workspace actions ŌĆö N captured. Say "show pending workspace actions".`
