# Superagent sources skill

The Sources contract is in `contracts/sources.md`. The agent uses this skill to read from the library.

## 1. Branch on intent

### List (`sources`)

1. Read `_memory/sources-index.yaml`.
2. Group by `category`, then within each by `kind` (documents first, then references).
3. Render a compact table:

   ```
   ## <category>
   | Title | Kind | Path | Last accessed | Reads |
   |---|---|---|---|---|
   ```

4. Filters:
   - `--category <name>`
   - `--kind document` / `--kind reference`
   - `--domain <id>`
   - `--project <id>`
   - `--asset <id>`
   - `--sensitive` (only sensitive)
   - `--unused` (read_count == 0)

### Search (`sources search "<query>"`)

1. Search across `_memory/sources-index.yaml.sources[]` `title` + `notes` + `tags`.
2. Plus a grep across `Sources/references/**/*.ref.md` body text (the notes section).
3. Plus a grep across `Sources/_cache/<*>/_summary.md`.
4. Rank by:
   - exact title match → top
   - `related_domain` matches user's recent context → boosted
   - recently accessed → boosted
5. Return top 10 with: title, kind, path, one-line excerpt of where the match was found.

### Fetch (`sources fetch <id>` or "what does <X> say?")

1. Resolve `<id>` from `_memory/sources-index.yaml`.
2. **For documents**: open the file directly. If small (< 4 KB) and text, surface inline. Else surface path + offer to open / extract specific pages.
3. **For references**: enter the local-first read flow per `contracts/sources.md` § 15.5:
   - Read the `.ref.md` (frontmatter + notes).
   - Compute `source_hash = sha256(kind + source)`.
   - Check `Sources/_cache/<hash>/_meta.yaml`.
   - **If cache fresh**:
     - Read `_summary.md` first. If the user's question is answered there, stop.
     - Else read `_toc.yaml` to find the relevant section.
     - Pull only the matching chunk(s) from `chunks/` or the matching range from `raw.<ext>`.
     - Update `_meta.yaml.last_used = now`; increment `_memory/sources-index.yaml.<row>.read_count`.
   - **If cache stale or missing**:
     - Invoke `tools/sources_cache.py fetch --ref <path>` which routes by `kind`:
       - `mcp` → calls the named MCP tool with the params.
       - `cli` → runs the shell command.
       - `url` → HTTP GET.
       - `api` → HTTP GET with auth from vault.
       - `file` → reads the local path.
       - `vault` → reads the vault item.
       - `manual` → asks the user to do it; aborts the skill.
     - Writes result to `_cache/<hash>/raw.<ext>`.
     - Generates `_summary.md` (AI-summarized, ≤ 500 words) — the agent does this in the same turn.
     - Generates `_toc.yaml` — sections + chunk index if `chunks/` was created.
     - Then jumps back to "cache fresh" path.

### Refresh (`sources refresh <id>` or `--refresh` flag)

Force a re-fetch even if cache is fresh. Useful when the upstream changed.

### Evict (`sources evict <id>` or `sources evict --all-stale` or `sources evict --over-cap`)

Manually clear cache entries. The cache is normally auto-evicted (TTL on read; LRU on write); this is for "I want to free up disk now" or "force re-fetch on next read".

`--all-stale`: evict everything with `last_used > ttl_minutes` ago.
`--over-cap`: bring cache under `config.preferences.sources.cache_max_mb` immediately (LRU).
`<id>`: evict just one.

### Coverage (`sources coverage`)

Report on what `Sources/` contains vs what's referenced from elsewhere:

- Domains with no Sources rows linked: candidate gap.
- Assets with no receipt / warranty document: candidate gap.
- Projects in `active` with no Sources: candidate gap.
- Documents listed in `documents-index.yaml` (the structured doc index) but not present in Sources: broken link.

Surfaces gaps; suggests `add-source` invocations to fill them.

## 2. Caching write rules

When fetching ANYTHING for the cache, the writer (`tools/sources_cache.py`) MUST:

1. Compute `sha256(kind + source)` deterministically (after canonicalizing `source` whitespace).
2. Create `Sources/_cache/<hash>/` if missing.
3. Write `raw.<ext>` (extension inferred from MIME type or kind).
4. Compute size; if `> config.sources.chunk_threshold_kb`, split into `chunks/` (~`chunk_target_kb` per chunk; respect natural boundaries — paragraphs, sections).
5. Generate `_summary.md` (LLM call; cap at 500 words).
6. Generate `_toc.yaml`.
7. Write `_meta.yaml`:

   ```yaml
   source_hash: "<hash>"
   source: { kind: "<kind>", source: "<source>", params: {...} }
   sensitive: <bool>
   fetched_at: <now>
   last_used: <now>
   ttl_minutes: <from .ref.md>
   size_bytes: <int>
   chunks_count: <int>
   ```

8. Run LRU eviction: while total `Sources/_cache/` size > `config.preferences.sources.cache_max_mb` * 1024 * 1024, delete the entry with oldest `last_used`. Never evict the entry being written in the same call.

## 3. Logging

Append a single line per `fetch`/`refresh`/`evict` to `interaction-log.yaml`:

```yaml
- timestamp: <now>
  type: source_access
  subject: "<id> (<fetch|refresh|evict>)"
  summary: "<source title> — cache <hit|miss|evicted>; size <kb>"
  related_domain: <id>
  related_project: <id>
```

For high-volume cache hits, batch the logging (one line per session, not per read) to avoid bloating `interaction-log.yaml`.
