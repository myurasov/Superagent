# Superagent personal-signals skill

This skill has two modes: **capture** (ambient, runs as a hook in any session) and **surface** (explicit, runs when the user asks).

## Capture mode (ambient)

Runs as a side-effect of every chat turn, not as a standalone invocation. Triggered by phrasing like:

- "I should…", "I keep…", "next time I should…", "I want to be more…"
- "<partner / family member> said I…"
- Pattern Superagent notices over multiple sessions / ingest runs.

When triggered:

1. Construct the row per `templates/memory/personal-signals.yaml` example:
   - `id`: `psig-YYYY-MM-DD-NNN`
   - `captured_at`: now
   - `category`: best-effort classification
   - `trigger_phrase`: short excerpt
   - `context`: 1-3 sentences
   - `suggested_practice`: optional
   - `source`: `self-reflection` | `partner-feedback` | `family-feedback` | `pattern-noticed`
   - `status`: `open`
2. Append to `_memory/personal-signals.yaml`.
3. Append a one-line tag at the END of the agent's reply (no preamble): `_Captured personal signal psig-NNN._`
4. **Do not** otherwise interrupt the conversation. The user can override capture at any time by saying "don't capture that" or by setting `config.preferences.personal_signals.capture_mode: "off"`.

## Surface mode (explicit)

User asked to see signals.

### List

1. Read `_memory/personal-signals.yaml`.
2. Filter by status (default: `open` + `practicing`).
3. Group by `category`.
4. Sort within each group by `captured_at` descending.
5. Render:

   ```
   ## Personal signals — surface

   ### <category>
   - **psig-NNN** [<status>] <trigger_phrase>
       Suggested practice: <suggested_practice>
       Source: <source>; captured <captured_at>; surfaced <surfaced_count> times before.
   ```

For each surfaced row, increment `surfaced_count`, set `last_surfaced: now`.

### Update

User picks a signal id and a new status:

- `practicing` — actively working on it.
- `internalised` — became a habit.
- `deferred` — not now.
- `discarded` — no longer relevant.

Update the row's `status` and (for `practicing`) `practiced_at: now`.

### Roll-up to Self domain

After any list / update, append (or update an existing) H4 entry in `Domains/Self/history.md`:

```markdown
#### <YYYY-MM-DD> — Personal-signals review

- **<category>**: <N open>, <K practicing>, <M internalised>, <P discarded>
- New since last review: <list of psig ids + trigger phrases>
- Status flips: <list>

---
```

The `Self` domain is the canonical home for self-development tracking; this skill is its primary writer.

## Privacy

- Capture is silent — only the one-line tag at end of reply.
- `_memory/personal-signals.yaml` is gitignored along with everything else.
- The user can disable capture entirely via `config.preferences.personal_signals.capture_mode: "off"`.
- The user can edit / delete rows manually any time.

## Logging

For surface invocations:

```yaml
- timestamp: <now>
  type: skill_run
  subject: "personal-signals (surface)"
  summary: "Surfaced <N> open signals across <K> categories."
  related_domain: "self"
```

Capture invocations do NOT log to `interaction-log.yaml` (the per-row tag in chat is the log). This keeps the log signal-rich.
