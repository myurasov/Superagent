# Superagent bills skill

## 1. Branch on intent

### List

User asked "what's due", "show bills", "next bill":

1. Read `_memory/bills.yaml`.
2. Recompute `next_due` for each row (handle late-paid skipped cycles).
3. Filter `status: active`.
4. Group:
   - **Overdue** (`next_due < today` AND no `history` entry for the current cycle).
   - **Due today**.
   - **Due in 1–7 days**.
   - **Due in 8–30 days**.
5. Within each group, sort by `next_due` ascending.
6. Format: `{name} {amount} {currency} — {next_due} ({auto-pay|manual})`.

If `--upcoming-only` requested, skip overdue. If `--domain <slug>`, filter accordingly.

### Mark-paid

User asked "mark <bill> paid", "I just paid <bill>":

1. Resolve bill by id or fuzzy match on name. Confirm if multiple match.
2. Ask for / infer payment date (default: today), amount paid (default: `amount`), confirmation number, source ("manual" or ingestor name), and **funding account** (default: `bills.yaml.<bill>.pay_from_account` when set; otherwise ask).
3. **Capture the confirmation artifact per `contracts/payment-confirmations.md`** — save the receipt / PDF / email / screenshot to the right `Sources/<Domain>/` (or `Projects/<project>/Resources/`) folder before writing the history row, so the next steps can record `confirmation_ref`.
4. Append to `bills.yaml.<bill>.history[]`:

   ```yaml
   - date: <payment-date>
     amount: <amount paid>
     confirmation: "<conf>"
     confirmation_ref: "<workspace-relative-path-to-saved-artifact>"
     source: "<source>"
     notes: ""
   ```
5. **Mirror to the funding account** per `contracts/payment-confirmations.md § 4 step 3a`. Append to `_memory/accounts-index.yaml.<account-id>.transactions[]`:

   ```yaml
   - id: "txn-<YYYYMMDD>-<NNN>"
     timestamp: <payment-date or full ISO datetime when known>
     direction: "out"
     amount: <amount paid>
     currency: "<currency, default USD>"
     counterparty: "<bill payee>"
     purpose: "<bill name> payment"
     channel: "<ach | card_debit | check | ...>"
     confirmation: "<conf>"
     confirmation_ref: "<same path as the bill-history row>"
     status: "pending"
     related_project: <project slug or null>
     related_bill: "<bill-id>"
     related_asset: []
     related_entities: ["bill:<bill-id>"]
     provenance:
       source: "<source>"
       at: <now>
     notes: ""
   ```

   Bump `accounts-index.yaml.<account-id>.last_updated`. Skip step 5 only when the bill has no funding account on file (cash, money order, etc.); record the reason in step 4's `notes`.
6. Recompute `next_due` from cadence + due_day relative to the payment date.
7. Update `bills.yaml.<bill>.last_updated`.

### Reconcile (`bills reconcile`)

User asked to reconcile recently-ingested transactions against open bills:

1. If a finance ingestor is enabled, read the recent transactions (Plaid / Monarch / CSV ingestor's normalized output — typically `_memory/transactions.yaml` or per-source `interaction-log.yaml` entries).
2. For each transaction in the trailing 30 days:
   - Match by `payee` substring + amount tolerance (±5%).
   - On match: append to the bill's `history[]` if not already present (dedup on `confirmation` or `(date, amount)` tuple).
3. Surface:
   - **Auto-matched**: count + bill names.
   - **Bills due in last 30 days with NO matching transaction**: list — these may have been missed.
   - **Transactions matching no bill**: list — candidates to add via `add-bill` or to flag as one-shot.
