# Superagent add-project skill

The full Projects contract is in `procedures.md` § 16. This skill is the front-end.

## 1. Collect the charter

Use `AskQuestion` for the multi-select fields; inline for the rest.

### Q1 — Name

Ask for the project name (e.g. "File 2026 taxes", "Italy summer trip", "Replace dishwasher", "Annual health tune-up 2026").

Auto-derive **slug** = lowercased, hyphenated, no punctuation, with a year suffix when the name implies a year. Examples:

- "File 2026 taxes" → `tax-2026`
- "Italy summer trip" → `italy-summer-2026` (auto-append current year if none in name)
- "Replace dishwasher" → `dishwasher-replace`
- "Annual health tune-up 2026" → `health-tuneup-2026`

Confirm the slug.

Check `_memory/projects-index.yaml.projects[]` and `.archived[]` for collisions; if a row with the same slug exists in `archived[]`, ask whether to **resume** (move it back) or **rename** the new one.

### Q2 — Goal (one sentence)

> "What does 'done' look like? One sentence."

Examples:
- "Federal + state returns filed; refund received OR balance paid."
- "Trip booked, packed, no flights or hotels missed."
- "Old dishwasher hauled away, new one installed and running through one cycle without leaks."

### Q3 — Scope

Two follow-up prompts:

> "What's IN scope?"
> "What's OUT of scope?"

Each accepts a multi-line list. The OUT-of-scope side is what saves you from scope creep three weeks in.

### Q4 — Target date

ISO 8601 date. Hard requirement to leave `planning` for `active`.

For recurring projects (taxes due April 15, year-end giving in December), the target_date for the first occurrence is the natural deadline; recurrence handles the next ones automatically.

### Q5 — Related domains (multi-select)

Present the 10 default domains plus any custom ones from `_memory/domains-index.yaml.domains[]` with `status: active`. The user picks 1+.

Examples:
- "File 2026 taxes" → `finance`
- "Italy summer trip" → `travel`, `family`
- "Replace dishwasher" → `home`
- "Annual health tune-up" → `health`
- "Renovate kitchen" → `home`, `finance`

### Q6 (optional) — Recurring

> "Does this project repeat? (none / annual / quarterly / monthly / every N years)"

For `every_n_years`, follow up with N.

### Q7 (optional) — Stakeholders

> "Who else is involved? (multi-select from contacts; or skip)"

Pulled from `_memory/contacts.yaml`.

### Q8 (optional) — Success criteria

> "List 3-5 checkable success criteria. (one per line; or skip and we'll work it out as we go)"

### Q9 (optional) — Deliverables

> "What concrete artifacts does this project produce? (e.g. 'filed return PDF', 'trip itinerary doc', 'dishwasher receipt + warranty in Sources'; or skip)"

### Q10 (optional) — Budget

> "Is there a planned spend? (amount + currency, or skip)"

## 2. Append the project row

Write to `_memory/projects-index.yaml.projects[]`:

```yaml
- id: "<slug>"
  name: "<name>"
  goal: "<goal>"
  scope: |
    In: <items>
    Out: <items>
  priority: "P2"            # default; can override later
  status: "planning"
  path: "Projects/<slug>"
  related_domains: <list>
  stakeholders: <contact-id list>
  start_date: <today>
  target_date: <target>
  completed_date: null
  recurring: "<recurring or 'none'>"
  recurring_n: <N or null>
  success_criteria: <list>
  deliverables: <list>
  budget: <{ planned, spent: 0, currency } or null>
  tags: []
  notes: ""
  created: <now ISO 8601>
  last_updated: <now ISO 8601>
```

## 3. Scaffold the folder

Create `workspace/Projects/<slug>/`:

- `info.md` — fill in template placeholders from the charter answers.
- `status.md` — initial RAG: `**Green** — newly chartered; gathering tasks`. Open / Done blocks empty (auto-rendered later).
- `history.md` — single H4 entry: `#### <today> — Project chartered` with the goal + scope summary.
- `rolodex.md` — empty templates; populated as the project takes touchpoints.

Write `sources.md` from `superagent/templates/projects/sources.md` (empty tables).
Do NOT pre-create `Resources/` or `Sources/` (lazy — created on first content).

## 4. Auto-create initial tasks (optional)

Ask:

> "Want me to seed initial tasks based on the charter? I'll suggest 3-5 and you can keep / edit / drop."

If yes: pattern-match the project kind for templated task lists:

| Project kind hint | Suggested tasks |
|---|---|
| Tax filing | "Gather W-2s + 1099s", "Compile charitable receipts", "Schedule call with accountant", "File return", "Archive return PDF in Sources/documents/taxes/<year>/" |
| Trip planning | "Book flights", "Book hotels", "Renew passport (if needed)", "Set up house-sitter / pet-sitter", "Build packing list", "Pre-trip TSA check" |
| Appliance replacement | "Get 3 quotes", "Read reviews", "Schedule install", "Coordinate haul-away of old", "Test on cycle", "File receipt + warranty in Sources/documents/warranties/" |
| Annual health tune-up | "Schedule annual physical", "Schedule dental cleaning", "Schedule eye exam", "Refill standing prescriptions", "Review labs from last year" |
| Job search | "Update resume", "Curate target list", "Reach out to 5 contacts", "Apply to N roles", "Schedule mock interviews" |
| Renovation | "Define scope + budget", "Get 3 contractor bids", "Get permits", "Schedule start", "Punch list at end" |

For each, create a `todo.yaml` row with `related_project: <slug>` and `priority: P2`. The user can re-prioritize after.

## 5. Cross-link Domain status

For each domain in `related_domains`:

- Append a one-line bullet to `Domains/<Domain>/status.md` § Next Steps:
  > "**Project**: <name> — target <target_date>."
- Append H4 entry to `Domains/<Domain>/history.md`:
  ```markdown
  #### <YYYY-MM-DD> — Project chartered: <name>

  Goal: <goal>. Target: <target_date>. See `Projects/<slug>/`.
  ```

## 6. Confirm
