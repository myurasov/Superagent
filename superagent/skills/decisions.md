# Superagent decisions skill

Implements ideas-better-structure.md item #24.

## 1. Branch on intent

### Capture (default for new decision)

User said something like "I just decided X" / "log a decision".

Collect (use `AskQuestion` for multi-select fields, inline for the rest):

1. **Decision** — one sentence ("Move to a high-yield savings account at ...").
2. **Context** — why was this decision needed?
3. **Alternatives considered** — list of strings (what was ruled out).
4. **Rationale** — why this option won.
5. **Confidence** — `high | medium | low`. Self-assessment.
6. **Reversibility** — `easy | medium | hard | one-way-door`.
7. **Related domain** (optional, from domains-index).
8. **Related project** (optional).
9. **Affects** — list of operational handles for entities the decision touches.
10. **Review at** — ISO 8601 date when to revisit and measure outcome.

Auto-derive **id** = `dec-<YYYY-MM-DD>-<NNN>`.

Append to `_memory/decisions.yaml.decisions[]`:

```yaml
- id: "<id>"
  ts: <now>
  decision: "<text>"
  context: "<text>"
  alternatives_considered: <list>
  rationale: "<text>"
  confidence: "<high|medium|low>"
  reversibility: "<easy|medium|hard|one-way-door>"
  related_domain: <id or null>
  related_project: <id or null>
  affects: <list of handles>
  source: "user"
  review_at: <date or null>
  outcome_measured_at: null
  outcome: null
  outcome_notes: ""
  revisited: []
  tags: []
```

Also append to events stream via `tools/log_window.py append --kind decision`.

### List

User said "show me decisions" / "decisions this week":

1. Read `_memory/decisions.yaml.decisions[]`.
2. Filter by window if specified (default: last 30 days).
3. Sort by `ts` desc.
4. Render compact table:
   ```
   | id | ts | decision | confidence | reversibility | review_at |
   |---|---|---|---|---|---|
   ```

### Review

User said "review decisions" / "what's due to review":

1. Filter rows where `review_at <= today` AND `outcome is null`.
2. For each: surface decision + context + rationale; ask:
   - **outcome** (`better | worse | neutral | inconclusive`).
   - **outcome_notes** (free text).
   - Optionally: **revisit?** (creates a new decision id pointing back to this one via `revisited: [<new_id>]`).
3. Append outcome to the existing row.

### Show

`decisions show <id>` — full detail view of one decision.

## 2. Surfacing

`weekly-review` reads decisions made in last 7 days and surfaces them under "## Decisions made this week".

`monthly-review` reads decisions where `review_at` falls in the prior month and surfaces them under "## Decisions due to review".
