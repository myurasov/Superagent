# Superagent weekly-review skill

Compute a **trailing 7-day window** ending today (local).

## 1. Run weekly-cadence ingestors

For sources with `schedule: weekly` in `data-sources.yaml`, run their ingestors (per the ingestion contract). Collect counts.

If `simplefin` is enabled (the canonical finance ingestor), run it explicitly:

```bash
uv run python -m superagent.tools.ingest.simplefin
```

## 2. Bookkeeper pass

If at least one finance ingestor is enabled and the recent transactions are mirrored:

1. Read transactions from `_memory/transactions.yaml` (SimpleFin / Plaid / Monarch / CSV all write here).
2. Run the reconciler for a 7-day window — it cross-references transactions against `bills.yaml` (matched / missed) and surfaces recurring-charge candidates not yet tracked in `bills.yaml` / `subscriptions.yaml`:

   ```bash
   uv run python -m superagent.tools.reconcile_transactions --days 7
   # or `--json` for structured consumption
   ```

3. Surface in the review under headings: **Bills missed**, **Bills paid**, **Recurring-charge candidates** (audit prompt: "Add to bills.yaml or subscriptions.yaml?").
4. Independent of the reconciler, total `inflows` and `outflows` for the trailing 7 days; group outflows by `category`; flag categories > 1.5× their 4-week trailing average.

If no finance ingestor is enabled, skip with a one-line note: *"Bookkeeper pass skipped — no finance ingestor configured. See `docs/data-sources.md#finance` to enable."*

## 3. Coach pass

If at least one fitness / health ingestor is enabled:

1. Workouts logged this week (count + total minutes + total distance if applicable).
2. Sleep average (hours, score) over last 7 nights.
3. Weight trend (delta vs 4 weeks ago).
4. Personal-signal events captured this week (count by category).
5. Streak status against any active goal in `Domains/Hobbies/status.md`.

If no fitness ingestor is enabled, surface manual reflection prompt:

> "Coach pass: no fitness ingestor configured. Want to log this week's workouts manually? (`Hobbies` domain)"

## 4. Concierge pass — coming up

Read `_memory/appointments.yaml`, `_memory/important-dates.yaml`, `_memory/documents-index.yaml`:

- Appointments in next 14 days.
- Important dates in next 30 days.
- Documents expiring in next 60 days.

Group by domain.

## 4b. Project rollup

Read `_memory/projects-index.yaml.projects[]`:

For each project with `status: active`:
- **Burn-down**: render `{checked}/{total} success criteria — {days} days to target` (when criteria ≥ 5).
- **This week's progress**: count of tasks completed for this project in last 7 days; H4 entries appended to project's `history.md` in last 7 days.
- **Stalled?**: if NO entry in the project's `history.md` AND no completed tasks in last 14 days → flag as `stalled`. Suggest pause or push.

Also list:
- **Projects in `planning`** — chartered but not yet active. Prompt: "ready to flip these to active?".
- **Projects with target_date in next 30 days** — will dominate the next monthly cycle.
- **Recently completed projects** (last 7 days) — surface lessons-learned prompt if not already captured.

## 5. Quartermaster pass — maintenance

Read `_memory/assets-index.yaml`:

- Maintenance items due in next 30 days.
- Warranties expiring in next 60 days.

Group by domain.

## 6. Tasks

Read `_memory/todo.yaml`:

- **Done this week** (count + list of titles).
- **Slipped** (open tasks with `due_date` in this week's window that aren't done).
- **New tasks created this week** (count by `source`).
- **Coming due in next 7 days** (P0 + P1).

## 7. Reflection prompt

Ask the user (one prompt, accept skip):

> "Two questions for the log — type a sentence each, or 'skip':
>
>   1. **What worked well this week?**
>   2. **What do you want to do differently next week?**"

Capture both answers as a single H4 entry in `Domains/Self/history.md`:

```markdown
#### <YYYY-MM-DD> — Weekly review

**Worked well**: <answer 1>

**Differently next week**: <answer 2>

---
```

If either answer hints at a personal signal (heuristic: contains words like "should", "need to", "want to be", "next time"), capture as a `personal-signals.yaml` row (per `contracts/capture.md` § 7.1) and tag it inline.

```

- Outliers: <flagged categories>
- New recurring charge candidates: <list with "add to bills/subs?">

## Coach

- Sleep avg: <hours>h, score <score>

- Streak status: <one line>
```

## 9. Update memory

3. If reflection captured a personal signal, append to `_memory/personal-signals.yaml`.
4. The `Domains/Self/history.md` H4 entry is already written in step 7.
