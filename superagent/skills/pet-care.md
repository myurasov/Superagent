# Superagent pet-care skill

## 1. Resolve the pet

Pets live in `contacts.yaml.contacts[]` with `role: pet` and a per-pet sub-folder under `Domains/Pets/<pet-slug>/`. (Per-pet sub-folders are created by `add-contact` when role is pet, with the same 4-file structure as a domain.)

If only one pet exists, default to it. Otherwise ask.

## 2. Branch on intent

### List due

For the resolved pet, surface:
- Next vet visit (from `appointments.yaml` filtered by `for_member: <pet>` AND `kind: vet`).
- Active medications and their next refill / dosage time (per-pet medication tracking — store in `Domains/Pets/<pet>/info.md` § Key Facts under a "Current meds" section).
- Vaccinations due (compute from a per-species cadence list — see § 4).
- Grooming due (if scheduled).
- Boarding planned (if any).
- Food rotation (next bag due).

### Log vet visit

Same as `appointments mark-complete`, with `for_member: <pet>` and `domain: pets`. Outcome appends to `Domains/Pets/<pet>/history.md`.

### Log vaccination

Ask: vaccine name, date, lot, vet, next due.

Append to a per-pet `vaccinations` block in `Domains/Pets/<pet>/info.md` § Key Facts:

```markdown
| Vaccine | Date | Next due | Vet | Lot |
|---|---|---|---|---|
| <vaccine> | <date> | <next> | <vet> | <lot> |
```

If `next due` is within 60 days, surface in `Domains/Pets/<pet>/status.md` § Next Steps.

### Log medication start / change / stop

Append to a per-pet `medications` block in `Domains/Pets/<pet>/info.md`. Track refill schedule similar to human meds.

### Log weight

Append to a per-pet `weights.csv` (created lazily) under `Domains/Pets/Resources/<pet>/`:

```
date,weight_lb,note
2026-04-28,12.4,"check-up"
```

Update `Domains/Pets/<pet>/info.md` § Key Facts current weight.

### Log food / treats

Append to a per-pet `food-log.md`. Useful for spotting "we switched brands and now she's scratching" patterns.

### Log boarding

Add an entry similar to a vet visit (provider = boarding facility from `contacts.yaml`). For multi-day stays, capture both drop-off and pickup events.

## 3. Sync downstream

- Update `Domains/Pets/status.md` § Recent Progress + Next Steps.
- Append to `interaction-log.yaml`.

## 4. Default vaccination schedules

| Species | Vaccine | Cadence (adult) |
|---|---|---|
| dog | rabies | 1-3 yr (per state) |
| dog | DHPP / DHLPP | 1-3 yr |
| dog | bordetella | annual |
| dog | leptospirosis | annual |
| cat | rabies | 1-3 yr |
| cat | FVRCP | 1-3 yr |
| cat | FeLV | annual (outdoor cats) |

These are conservative defaults; the actual schedule should match what the vet recommends. The skill is a reminder, not medical advice.
