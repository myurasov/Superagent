## Summary

This migration introduces the `workspace/.version` file used by every
subsequent migration. Pre-0.2.0 workspaces have no version tracking;
this migration creates the file and stamps it with `0.2.0`. No other
workspace data changes. Fully reversible (delete the file).

## Pre-flight checks

- [ ] The configured workspace path exists and is a directory.
- [ ] `workspace/_memory/` exists (i.e. the workspace was scaffolded by
      `init` at some point — we are not migrating an empty directory).
- [ ] `workspace/.version` does NOT already exist (the migration would
      be a no-op; the skill should have skipped this step).

## Migrate

1. Resolve the workspace path: read `workspace/_memory/config.yaml.preferences.workspace_path`
   if set; otherwise default to `./workspace/`.
2. Write the file `<workspace>/.version` containing exactly:

   ```
   0.2.0
   ```

   (One line, trailing newline, UTF-8 encoded, no BOM.)

   Implementation: `tools/version.py.set_workspace_version(workspace, "0.2.0")`.

## Validate

- [ ] `<workspace>/.version` exists and is a regular file.
- [ ] `tools/version.py.workspace_version(workspace)` returns the
      string `"0.2.0"`.
- [ ] No other files under `workspace/` were modified by this
      migration (compare mtimes if uncertain).

## Revert

1. Delete `<workspace>/.version`.

After revert, `tools/version.py.workspace_version(workspace)` falls back
to the documented default `"0.1.0"` per `contracts/versioning.md` § 2.
