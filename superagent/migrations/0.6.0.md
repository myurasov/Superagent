## Summary

Adds the `simplefin` ingestor (`superagent/tools/ingest/simplefin.py`),
the one-time setup-token claimer (`superagent/tools/simplefin_claim.py`),
and the read-only transaction reconciler
(`superagent/tools/reconcile_transactions.py`). Together they pull
bank / credit-card / brokerage transactions into
`_memory/transactions.yaml`, cross-reference them against `bills.yaml`
and `subscriptions.yaml`, and surface recurring-charge candidates not
yet tracked.

The release also wires the simplefin ingestor + reconciler into the
weekly-review skill's Bookkeeper pass (step 2), documents
`simplefin_account_id` as a per-account cross-link in
`accounts-index.yaml`, and ships a one-row registry entry in
`tools/ingest/_registry.py`.

This migration touches workspace data **only by bumping `.version`**.
`transactions.yaml` and `_memory/sensitive/simplefin-credentials.yaml`
are created lazily by the ingestor on first run; users who never enable
SimpleFin are unaffected.

## Revert

1. Write `workspace/.version` back to `0.5.0`. Implementation:
   `tools/version.py.set_workspace_version(workspace, "0.5.0")`.

After revert, the framework files added by this release
(`tools/ingest/simplefin.py`, `tools/simplefin_claim.py`,
`tools/reconcile_transactions.py`, the registry entry, the
weekly-review hook) remain on disk but are inert if the user has
not generated SimpleFin credentials. If the user wants a clean
revert, they may additionally:

- Remove `workspace/_memory/sensitive/simplefin-credentials.yaml`
  (the long-lived Access URL).
- Remove `workspace/_memory/transactions.yaml` (the ingested rows).
- Remove the `simplefin` row from `workspace/_memory/data-sources.yaml`.

None of those removals are required for a clean version revert.

## Notes

- `accounts-index.yaml` gains an optional `simplefin_account_id` field
  on per-account rows, used to cross-link ingested transactions back
  to their account row. Existing rows without the field continue to
  work; new rows added by the user can populate it from the SimpleFin
  account UUID (visible at `bridge.simplefin.org` under Accounts).
- The reconciler is read-only; it never mutates `bills.yaml`,
  `subscriptions.yaml`, or `transactions.yaml`. Its output is
  consumed by the weekly-review skill.
- SimpleFin's `/accounts` endpoint warns when the requested window
  exceeds 45 days. The ingestor chunks larger windows accordingly
  (`SIMPLEFIN_WINDOW_DAYS = 45`). The 90-day figure in SimpleFin's
  developer guide is the hard server-side cap; 45 days is the warn
  threshold.
